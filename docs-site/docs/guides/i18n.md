# Internationalization

skellydocs provides a `defaultLocales()` helper that generates a Docusaurus i18n config with human-readable directory names.

## Using defaultLocales()

```typescript
import { defaultLocales } from '@freemocap/skellydocs';

const config: Config = {
  // ...
  i18n: defaultLocales(),
};
```

This generates the following config:

```typescript
{
  defaultLocale: 'en',
  locales: ['en', 'es', 'ar', 'zh-CN'],
  localeConfigs: {
    es: { label: 'Español', htmlLang: 'es', path: 'es-spanish' },
    ar: { label: 'العربية', htmlLang: 'ar', direction: 'rtl', path: 'ar-arabic' },
    'zh-CN': { label: '简体中文', htmlLang: 'zh-CN', path: 'zh-chinese' },
  },
}
```

## Starter locales

The four starter locales were chosen to test the key rendering edge cases:

| Locale | Directory | Why |
|---|---|---|
| English | (default) | Base language, source of truth |
| Spanish (`es`) | `i18n/es-spanish/` | Latin script, LTR — simplest translation case |
| Arabic (`ar`) | `i18n/ar-arabic/` | Right-to-left (RTL) layout |
| Chinese (`zh-CN`) | `i18n/zh-chinese/` | Character-based rendering |

## Human-readable directory names

Standard Docusaurus uses locale codes as directory names (`i18n/es/`, `i18n/ar/`). When you're scanning a file tree with many locales, `es-spanish` is immediately identifiable while `es` requires you to remember which language that code maps to.

Docusaurus's `path` config option maps these readable names to standard locale codes, so URLs stay clean (`/es/docs/intro`, not `/es-spanish/docs/intro`) and HTML `lang` attributes stay valid.

## Translation workflow

There are two types of translatable content:

**Short UI strings** (~20 keys) — hero title, tagline, feature card titles, button labels. These use Docusaurus's `<Translate>` component and live in `code.json` files per locale.

**Doc content** — all substantial content stays in markdown/MDX files. Each locale gets a full translated copy in `i18n/{locale}/docusaurus-plugin-content-docs/current/`.

A `translate` CLI subcommand for LLM-powered translation is planned but not yet implemented.

## Adding more locales

Override the `i18n` config in your `docusaurus.config.ts` to add or remove locales. The `defaultLocales()` return value is just a plain object — spread it and modify:

```typescript
const i18n = defaultLocales();
i18n.locales.push('fr');
i18n.localeConfigs.fr = {
  label: 'Français',
  htmlLang: 'fr',
  path: 'fr-french',
};
```
