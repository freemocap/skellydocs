# GitHub Issues Integration

skellydocs connects your documentation to GitHub issues in two ways:

1. **Feature cards** on the landing page show linked issues
2. **Roadmap page** fetches and displays all issues from your repo

## Linking issues to feature cards

In your `content.config.tsx`, each feature has an `issues` array. Each entry is a `LinkedIssue` with a `label` and a GitHub `url`:

```tsx
features: [
  {
    id: 'frame-sync',
    icon: 'đź”’',
    title: 'Frame-Perfect Sync',
    description: 'Keeps all cameras in lock-step.',
    summary: <>...</>,
    issues: [
      {
        label: 'Hardware synchronization (external trigger)',
        url: 'https://github.com/freemocap/skellycam/issues/1',
      },
      {
        label: 'Target frame rate setting',
        url: 'https://github.com/freemocap/skellycam/issues/2',
      },
    ],
    docPath: 'core/frame-sync',
  },
],
```

Each feature card displays a collapsible "Linked Issues" section. When expanded, the component:

- Fetches metadata from the GitHub API (type, status, labels)
- Shows an **Issue** or **PR** badge
- Shows an **open** or **closed** status indicator
- Displays any GitHub labels as colored chips
- Caches results in `localStorage` for 5 minutes

If the GitHub API is unreachable, it falls back to displaying just the label text.

### Guarantee issues

You can also link issues to the guarantees section:

```tsx
guarantees: [
  'All recordings have <strong>identical frame counts</strong>',
],

guaranteeIssues: [
  {
    label: 'Crash-safe recordings',
    url: 'https://github.com/freemocap/skellycam/issues/12',
  },
],
```

## Setting up the roadmap

The roadmap page at `/roadmap` fetches issues directly from GitHub's API. It's powered by two things:

### 1. The `roadmap.tsx` page

This file is generated by the CLI:

```tsx
import { RoadmapPage, collectLinkedUrls } from '@freemocap/skellydocs';
import config from '../../content.config';

const REPO = 'freemocap/skellycam';

export default function Roadmap() {
  return <RoadmapPage repo={REPO} pinnedIssues={collectLinkedUrls(config)} />;
}
```

- `repo` â€” your GitHub `org/repo` slug. The component fetches issues from `https://api.github.com/repos/{repo}/issues`
- `pinnedIssues` â€” issue URLs from your `content.config.tsx` that should be pinned at the top of the roadmap

### 2. The `"roadmap"` label

The roadmap fetches issues that have the **`roadmap`** label on your GitHub repo. To add an issue to your roadmap:

1. Go to your GitHub repo's Issues tab
2. Create or open an issue
3. Add the `roadmap` label to it

The label is case-sensitive â€” it must be exactly `roadmap`.

### 3. Pinned issues via `collectLinkedUrls`

`collectLinkedUrls(config)` scans your `content.config.tsx` and collects all issue URLs from `features[].issues` and `guaranteeIssues`. These issues appear on the roadmap **even if they don't have the `roadmap` label**.

This means any issue you link to a feature card automatically shows up on the roadmap too.

## Roadmap features

The roadmap page provides:

- **Search** â€” filter by title or description
- **Type filter** â€” show all, issues only, or PRs only
- **Status filter** â€” show all, open only, or closed only
- **Sort** â€” by recently updated, newest, or oldest
- **View toggle** â€” grid view (two columns) or list view (single column)
- **ETag caching** â€” responses are cached with ETags to minimize API calls

### About this roadmap

The roadmap includes a collapsible "About this roadmap" section that explains to visitors how the roadmap works and how they can contribute.

## GitHub API rate limits

The roadmap and linked issues use GitHub's **unauthenticated** API, which is rate-limited to 60 requests per hour per IP. The components mitigate this with:

- **ETag-based caching** â€” if the data hasn't changed, GitHub returns `304 Not Modified` and doesn't count against the rate limit
- **localStorage caching** â€” responses are cached locally with a 5-minute TTL
- **Graceful degradation** â€” if the API is unreachable, the roadmap shows an error with a retry button, and linked issues show their label text without metadata

For most documentation sites, 60 requests/hour is more than enough. If you hit rate limits during development, wait a few minutes or check that your browser's localStorage cache is working.

## Summary

| Feature | How issues get there | What's displayed |
|---|---|---|
| Feature cards | `issues` array in `content.config.tsx` | Label, type badge, status dot, GitHub labels |
| Guarantee section | `guaranteeIssues` array in `content.config.tsx` | Same as feature cards |
| Roadmap | `roadmap` label on GitHub + pinned issues from config | Full card with title, excerpt, badges, labels, date |
