# Tutorial

This walkthrough covers the full setup and customization of a skellydocs site. If you just want to get running quickly, see the [Getting Started](getting-started) guide.

## Scaffolding

Run the CLI scaffolder:

```bash
npx @freemocap/skellydocs init
```

After answering the prompts, you'll have a complete Docusaurus site. The generated files fall into two categories:

**Files you edit** — your project's content:
- `content.config.tsx` — landing page content (hero, features, guarantees)
- `docs/*.mdx` — documentation pages
- `blog/*.mdx` — blog posts
- `static/img/` — images and logos

**Files you rarely touch** — wiring that delegates to the theme:
- `docusaurus.config.ts` — Docusaurus configuration (already wired to skellydocs CSS)
- `src/pages/index.tsx` — one-liner wrapper around `IndexPage`
- `src/pages/roadmap.tsx` — one-liner wrapper around `RoadmapPage`
- `sidebars.ts` — auto-generates sidebar from file structure

## Understanding content.config.tsx

This is the heart of your landing page. It's a typed config object that drives the `IndexPage` component.

### Hero section

The hero appears at the top of your landing page:

```tsx
hero: {
  title: 'SkellyCam',
  accentedSuffix: 'Cam',
  subtitle: 'The camera backend for FreeMoCap',
  tagline: 'Frame-perfect multi-camera synchronization for USB webcams',
  logoSrc: '/skellycam/img/skellycam-logo.svg',
  parentProject: {
    name: 'FreeMoCap',
    url: 'https://freemocap.org',
  },
},
```

- `title` — the main heading
- `accentedSuffix` — the part of the title rendered in the accent color (`--sk-accent`)
- `logoSrc` — path to your logo in `static/`. Must include the `baseUrl` prefix

### Features

Each feature becomes a card on the landing page that links to a doc page:

```tsx
features: [
  {
    id: 'frame-sync',
    icon: '🔒',
    title: 'Frame-Perfect Sync',
    description: 'Keeps all cameras in lock-step.',
    summary: (
      <>
        A <Tip shortInfo="Gated on relative frame counts">
          frame-count-gated capture protocol
        </Tip> ensures all cameras stay in lock-step.
      </>
    ),
    issues: [
      {
        label: 'Hardware synchronization',
        url: 'https://github.com/freemocap/skellycam/issues/1',
      },
    ],
    docPath: 'core/frame-sync',
  },
],
```

- `summary` supports full TSX — you can use `Tip`, `<strong>`, links, etc.
- `issues` are displayed as a collapsible "Linked Issues" section on the card (see [GitHub Issues Integration](github-issues))
- `docPath` is the route to the feature's documentation page (relative to `/docs/`)

### Guarantees

Guarantees appear below the feature cards as a checklist:

```tsx
guarantees: [
  'All recordings have <strong>identical frame counts</strong>',
  'Each payload contains <strong>exactly one image per camera</strong>',
],

guaranteeIssues: [
  {
    label: 'Crash-safe recordings',
    url: 'https://github.com/freemocap/skellycam/issues/12',
  },
],
```

Guarantee text supports inline HTML (`<strong>`, `<em>`, etc.).

## Writing documentation pages

All docs use the `.mdx` extension, which lets you import and use React components inline.

### Page structure

```mdx
---
sidebar_position: 1
title: My Page Title
---

import { AiGeneratedBanner, Tip } from '@freemocap/skellydocs';

<AiGeneratedBanner />

# My Page Title

Regular markdown content with
<Tip shortInfo="A brief explanation" longInfo="A longer, more detailed explanation">
  inline tooltips
</Tip>.
```

### Using the Tip component

The `Tip` component supports two modes:

**Legacy mode** — simple hover tooltip:

```mdx
<Tip text="Explanation shown on hover">highlighted text</Tip>
```

**Two-stage mode** — hover for short info, click to expand:

```mdx
<Tip
  shortInfo="Brief explanation shown on hover"
  longInfo="Detailed explanation revealed on click"
  href="https://example.com/learn-more"
>highlighted text</Tip>
```

- Hover shows `shortInfo` in a tooltip
- "Click to learn more" expands the tooltip to reveal `longInfo`
- Optional `href` adds a "Learn more" link at the end

### Using CoreFeatureHeader

For dedicated feature doc pages, use `CoreFeatureHeader` at the top:

```mdx
import { CoreFeatureHeader } from '@freemocap/skellydocs';

<CoreFeatureHeader
  feature={{
    id: 'frame-sync',
    icon: '🔒',
    title: 'Frame-Perfect Sync',
    description: 'Keeps all cameras in lock-step.',
    summary: <>...</>,
    issues: [],
    docPath: 'core/frame-sync',
  }}
  repoUrl="https://github.com/freemocap/skellycam"
/>
```

This renders the feature's icon, title, description, and linked issues in a consistent header block.

## Customizing the theme

### Accent color

Override `--sk-accent` in a custom CSS file to change your site's accent color:

```css
:root {
  --sk-accent: #60a5fa;  /* blue instead of the default green */
  --sk-accent-dim: rgba(96, 165, 250, 0.15);
}
```

Add your CSS file to the `customCss` array in `docusaurus.config.ts`:

```ts
theme: {
  customCss: [
    require.resolve('@freemocap/skellydocs/css/custom.css'),
    require.resolve('./src/css/custom.css'),  // your overrides
  ],
},
```

### Available design tokens

| Token | Default | Purpose |
|---|---|---|
| `--sk-bg-deep` | `#06050e` | Page background |
| `--sk-bg-surface` | `#0e0c1a` | Card/surface background |
| `--sk-border` | `#1a1730` | Borders and dividers |
| `--sk-text` | `#e8e6f0` | Primary text |
| `--sk-text-dim` | `#b0aec3` | Secondary/muted text |
| `--sk-accent` | `#6ee7b7` | Accent color |
| `--sk-purple` | `#a78bfa` | Secondary accent (links, badges) |

## Updating skellydocs

When a new version of `@freemocap/skellydocs` is released:

```bash
npm install @freemocap/skellydocs@latest
```

All theme components, CSS, and styles update automatically. New component props are always backward compatible — existing sites continue to work without changes.
